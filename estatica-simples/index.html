SI-1: Design de Software

Descrição

Sistema de Informação I é uma disciplina obrigatória do curso de ciências da computação da UFCG, voltada para o desenvolvimento web e técnicas para melhorar o desenvolvimento de software.

Objetivos

No final do curso, o aluno deve ser capaz de:

  1. Descrever os objetivos do projeto de software e o papel no desenvolvimento de um software.
  2. Definir o que são padrões de projeto e nomear um conjunto de padrões.
  3. Criticar a necessidade e/ou oportunidade da aplicação de padrões em um contexto.
  4. Descrever e criticar uma arquitetura de software.
  5. Aplicar boas práticas e padrões de projeto na transformação de uma especificação em um projeto flexível e reutilizável.
  6. Produzir um software complexo utilizando boas práticas e padrões de projeto.
  7. Produzir um software web que utilize tecnologias atualmente populares no mercado.
  8. Conectar o software sendo implementado às necessidades do cliente que o demanda.
  9. Criar software multithreaded

Conteúdo

Os principais conteúdos que serão desenvolvidos na disciplina:

  1. Principios de Design: Modularidade, Coesão e Acoplamento.
  2. Paradigmas de Design: Funcional, Orientado a objetos e mais...
  3. Padrões de projeto: Design Patterns.
  4. Noções de arquitetura de sotfware.

Avaliação

Há 3 tipos de avaliação: minitestes, labs e um projeto.

  • Minitestes: Peso 35%

    Acontecem em qualquer aula e o aluno pode perder 25% dos minitestes e não haverá reposição de minitestes

  • Labs: Peso 30 %

    São atividades de programação relativamente pequenas que visam exercitar os conceitos vistos no curso e as tecnologias necessárias para o desenvolvimento do projeto.

  • Projeto: Peso 35%

    É desenvolvido em grupo e composto e composto de três milestone (com pesos iguais na nota final)
